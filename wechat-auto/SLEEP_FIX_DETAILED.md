# Mac防睡眠详细指南

## 🚨 为什么必须防睡眠？

**问题**：电脑睡眠/关屏后，OpenClaw Gateway进程挂起，微信自动化测试中断。

**解决方案**：彻底禁用睡眠和显示器休眠。

## 🔧 三种执行方案

### 方案1：终端命令（最简单）
```bash
# 打开终端（应用程序 → 实用工具 → 终端）
sudo pmset -a sleep 0 displaysleep 0
```
**执行步骤**：
1. 复制上面命令
2. 粘贴到终端，按回车
3. 输入电脑密码（输入时不显示字符，正常输入后按回车）

### 方案2：系统设置（图形界面）
1. **系统设置** → **电池**
2. **电源适配器**标签：
   - 关闭显示器：从不
   - 防止自动睡眠：勾选
3. **锁定屏幕**：
   - 关闭显示器：从不
   - 防止自动睡眠：勾选

### 方案3：脚本一键执行
```bash
# 在终端执行
bash ~/.openclaw/workspace/wechat-auto/fix_sleep.sh
```

## 🧪 验证防睡眠生效

### 测试方法
1. **执行命令后**，等待30秒
2. **手动关屏**（按电源键或合上笔记本）
3. **等待2-3分钟**
4. **用手机飞书发消息**测试

### 成功标志
- ✅ 关屏后手机发消息，我能秒回
- ✅ 电脑不进入睡眠状态
- ✅ 网络连接保持正常

## ⚠️ 常见问题解决

### 问题1：命令提示"Operation not permitted"
**原因**：系统完整性保护限制
**解决**：
```bash
# 重启电脑，按Command+R进入恢复模式
# 打开终端，执行：
csrutil disable
# 重启电脑，再执行防睡眠命令
```

### 问题2：输入密码后无反应
**原因**：密码输入正确但终端不显示
**解决**：正常输入密码后按回车，即使不显示字符

### 问题3：命令执行成功但睡眠仍在
**原因**：其他程序覆盖设置
**解决**：
```bash
# 查看当前所有设置
pmset -g
# 检查是否有其他配置覆盖
```

### 问题4：合上笔记本后仍睡眠
**原因**：合盖睡眠是独立设置
**解决**：
```bash
# 禁用合盖睡眠
sudo pmset -a lidwake 1
sudo pmset -a acwake 1
```

## 📊 当前状态检查

### 检查命令
```bash
# 查看当前电源设置
pmset -g | grep -E "sleep|displaysleep"
```

### 期望输出
```
 sleep         0
 displaysleep  0
```

### 如果输出不同
```bash
# 强制重新设置
sudo pmset -a sleep 0
sudo pmset -a displaysleep 0
sudo pmset -a hibernatemode 0
```

## 🔄 恢复默认设置

### 测试完成后恢复
```bash
# 恢复默认设置
sudo pmset -a sleep 1
sudo pmset -a displaysleep 10
```

### 临时测试方案
如果不想永久修改，可以：
1. 执行防睡眠命令
2. 进行微信自动化测试
3. 测试完成后立即恢复

## 🎯 微信自动化测试前提

### 必须满足的条件
1. ✅ 防睡眠设置生效
2. ✅ 微信测试小号登录
3. ✅ 屏幕不锁屏
4. ✅ 网络连接稳定

### 测试顺序
1. **先验证防睡眠**：关屏测试消息收发
2. **再测试微信自动化**：运行测试脚本
3. **最后业务测试**：模拟贷款咨询回复

## 💡 最佳实践

### 安全建议
1. **测试期间保持电脑供电**（插电源）
2. **关闭屏幕保护程序**
3. **禁用自动锁定**
4. **保持网络连接**（建议有线网络）

### 故障排查顺序
1. 检查防睡眠设置 → `pmset -g`
2. 测试网络连接 → `ping 8.8.8.8`
3. 测试OpenClaw状态 → `ps aux | grep openclaw`
4. 测试消息收发 → 关屏后手机发消息
